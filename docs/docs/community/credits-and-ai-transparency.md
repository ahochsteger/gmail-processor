# Credits and AI Transparency

We highly value community involvement and aim to be transparent about the development process of Gmail Processor.

## Proper Attribution of Community Contributions

Community involvement in any way is more important than the final code that is checked into the repository. We want to make sure contributions from developers other than the maintainers are mentioned where it makes sense (e.g. PR, release notes) and kudos is given to them.
This applies even if it's just an idea, a suggestion, documentation, or anything else in an issue or discussion, and also if the originally suggested implementation is modified or re-implemented from scratch.

To ensure proper attribution is given:

1. **Commit Message Trailers:** Maintainers and contributors should use the `Co-authored-by:` trailer in commit messages to credit the person who suggested the idea or provided the initial code.
    ```
    Co-authored-by: Name <name@example.com>
    ```
    If only the GitHub username is known:
    ```
    Co-authored-by: Username <Username@users.noreply.github.com>
    ```
2. **Issue/PR References:** Mention the issue or discussion in the commit message (e.g., `closes #123` or `resolves #123`). Tools like `release-please` will automatically link to these in the generated release notes.
3. **All-Contributors:** The project uses the [all-contributors](https://allcontributors.org/) specification. If you see a contribution (code, idea, bug report, etc.), feel free to ask the maintainer or a bot to add the person to the README.
    ```
    @all-contributors please add @username for ideas, code, bug
    ```
    Alternatively, run `npm run update:contributors` and use `npx all-contributors add <username> <contribution>` locally before pushing.

## Transparent AI-Assisted Development

While using AI for tedious repetitive work, the human must stay in the loop to be responsible for what goes into the repository. It is not about giving the AI agent credit for its work, but to make transparent where and what has been done AI-assisted. This ensures we don't give the impression that everything is done by the author alone.

### How to Declare AI Assistance

1. **Commit Trailers:** Use a standard trailer like `AI-Assisted: true` in your commit message.
    ```
    feat: some new feature

    This commit introduces a new feature based on the discussion in #456.

    AI-Assisted: true
    ```
2. **Pull Request Descriptions:** Add a short sentence in the PR description, for example:
    * "This PR was created with AI assistance for code generation."
    * "The test suite in this PR was written with the help of AI."
3. **Comments (if applicable):** If a very specific, complex algorithm was generated by AI, a short inline comment could be added, though trailers and PR descriptions are generally preferred to avoid cluttering the code.

By following these guidelines, we ensure credit is given where it's due and that the use of AI tools is declared transparently.
